Discuss the scope of var, let and const

  • var:

    1.we can be re-declared and re-assigned it.

    2.var declarations are hoisted and sometimes it display unexpected behavior.

  • let:

    1.It is block-scoped.

    2.let declaration are not hoisted.

  • const:

    1.we can not re-assigned after declaration.

    2.It is also block-scoped.

    • Use cases of null and undefined

  • null:

    1.Null is used when we want to explicitly indicate the absence of a value or to reset a variable value.

    2.It can be used to check if a value is intentionally missing or unset.

  • undefined:

    1.Undefined is the default value of variables that have been declared but not initialized.

    2.If a function parameter is not provided with a value it is undefined by default.

    • Understanding REST API

  • REST: REST (Representational State Transfer) isan architectural style for designing networked applications.

    1.Statelessness:Each request from a client to a server must contain all the information needed to understand and process the request as there is no session state stored on the server between request.

    2.HTTP Methods:REST APIs use standard HTTP methods such as get, post, put and delete to perform operations on resources.

    3.Resource-Based:Resources are identified by unique urls and interactions with these resources are performed using HTTP methods.

    4.Representations: Resources can have different representations and clients can request the representation they prefer.
